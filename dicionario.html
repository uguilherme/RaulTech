Dicionário de Engenharia de Software

Termos

Agile

Agile é uma abordagem de desenvolvimento de software que enfatiza a colaboração, iteração e entrega rápida de software funcional.

SCRUM

O SCRUM é um framework ágil amplamente utilizado para gerenciar projetos de desenvolvimento de software. Ele é baseado em iterações chamadas de "sprints".

UML

A UML (Unified Modeling Language) é uma linguagem de modelagem amplamente usada para visualizar, especificar, construir e documentar sistemas de software.

Teste de Unidade

O teste de unidade é uma prática de teste de software em que unidades individuais ou componentes do código são testados de forma isolada para verificar sua funcionalidade.

Código Limpo

Código limpo refere-se à escrita de código que é fácil de ler, entender e manter. Isso ajuda a melhorar a qualidade do software e a colaboração na equipe.

Requisitos de Software

Requisitos de software são as funcionalidades, características e restrições que um sistema de software deve atender para satisfazer as necessidades dos usuários e das partes interessadas.

Design Patterns

Design patterns são soluções reutilizáveis para problemas comuns de design de software. Eles ajudam a promover a reutilização e a manutenção do código.

Ciclo de Vida de Desenvolvimento de Software

O ciclo de vida de desenvolvimento de software é o processo pelo qual o software é concebido, projetado, desenvolvido, testado, implantado e mantido ao longo do tempo.

Engenheiro de Software

Um engenheiro de software é um profissional que projeta, desenvolve, testa e mantém software. Eles aplicam princípios de engenharia ao desenvolvimento de software.
